Before testing OAuth, you need to configure OAuth apps in Google Console and GitHub, then update your .env file with the client IDs and secrets.

Google OAuth Setup

  1. Go to Google Cloud Console
  2. Create a new project or select existing
  3. Enable Google+ API
  4. Create OAuth 2.0 credentials
  5. Add authorized redirect URI: http://localhost:8080/auth/oauth/google/callback
  6. Update .env with your client ID and secret

GitHub OAuth Setup

  1. Go to GitHub Developer Settings
  2. Create a new OAuth App
  3. Set Authorization callback URL: http://localhost:8080/auth/oauth/github/callback
  4. Update .env with your client ID and secret

Regular Authentication

OAuth doesn't replace regular authentication - you can still:

  • Register with email/password
  • Login with username/password
  • Use MFA/TOTP authentication
  • Refresh tokens and logout normally

API Endpoints

  • GET /auth/oauth/google - Start Google OAuth flow
  • GET /auth/oauth/github - Start GitHub OAuth flow
  • GET /auth/oauth/google/callback - Google OAuth callback
  • GET /auth/oauth/github/callback - GitHub OAuth callback
  • GET /auth/oauth/failure - OAuth failure handler
